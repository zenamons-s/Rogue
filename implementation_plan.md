# План реализации Rogue-like игры на Go

## Общие принципы
- Следовать чистой архитектуре с разделением на слои: Domain, Presentation, Data.
- Использовать идиоматичный Go: пакеты, интерфейсы, тесты.
- Поэтапная разработка с приоритетом на core-механики.

## Этап 0: Подготовка окружения
1. Инициализировать проект Go: `go mod init rogue-game`.
2. Добавить зависимости: `tcell` для UI, `testify` для тестов.
3. Настроить структуру каталогов согласно архитектуре.
4. Создать Makefile для сборки, тестов, линтеров.

## Этап 1: Domain Layer - Сущности (Задание 1)
1. Определить структуры данных в пакете `entities`:
   - `GameSession`, `Level`, `Room`, `Corridor`
   - `Character` с полями: MaxHealth, Health, Dexterity, Strength, CurrentWeapon
   - `Backpack` с слотами для предметов
   - `Enemy` с типом, здоровьем, ловкостью, силой, враждебностью
   - `Item` с типом, подтипом, модификаторами
2. Определить перечисления (enum) для типов врагов, предметов, etc.
3. Написать unit-тесты для проверки корректности структур.

## Этап 2: Domain Layer - Генерация уровней (Задание 3)
1. Реализовать пакет `generation`:
   - Генерация 9 комнат на уровне (3x3 сетка).
   - Случайные размеры и позиции комнат в пределах секторов.
   - Генерация коридоров, соединяющих комнаты, гарантируя связность.
   - Проверка связности графа комнат (алгоритм DSU или BFS).
2. Генерация стартовой комнаты и комнаты выхода.
3. Генерация врагов и предметов в комнатах (кроме стартовой).
4. Масштабирование сложности в зависимости от номера уровня.

## Этап 3: Domain Layer - Игровая механика (Задание 2)
1. Пакет `gameplay`:
   - Логика перемещения персонажа по клеткам.
   - Логика боя (turn-based, проверка попадания, урон).
   - Логика использования предметов (еда, зелья, свитки, оружие).
   - Логика инвентаря (подбор, использование, ограничение по слотам).
   - Логика врагов (типы, поведение, преследование).
2. Реализовать формулы для характеристик:
   - Шанс попадания на основе ловкости.
   - Урон на основе силы и оружия.
   - Восстановление здоровья от еды.
   - Временные/постоянные баффы от зелий и свитков.
3. Реализовать систему ходов: игрок -> враги.

## Этап 4: Data Layer - Сохранение и загрузка (Задание 5)
1. Пакет `storage`:
   - Определить интерфейсы репозиториев (`GameRepository`, `StatsRepository`).
   - Реализовать JSON-сериализацию структур Domain.
   - Сохранение текущей игры в файл `save.json`.
   - Загрузка игры из файла с восстановлением состояния.
   - Ведение статистики и таблицы рекордов в `scoreboard.json`.
2. Обеспечить целостность данных при сохранении/загрузке.

## Этап 5: Presentation Layer - Рендеринг (Задание 4)
1. Пакет `render` с использованием `tcell`:
   - Отрисовка карты: стены, пол, коридоры.
   - Отрисовка сущностей: игрок, враги, предметы.
   - Интерфейс: панель статуса, инвентарь, меню.
   - Туман войны (Fog of War) на основе Ray Casting / Bresenham.
2. Пакет `ui`:
   - Меню: новая игра, загрузка, таблица рекордов, выход.
   - Управление: WASD для движения, клавиши для использования предметов.
   - Выбор предметов из инвентаря (цифры 1-9).
3. Реализовать 2D вид с мини-картой.

## Этап 6: Integration - Сборка игрового цикла
1. Пакет `app`:
   - Координация слоёв: инициализация, запуск игрового цикла.
   - Обработка ввода и делегирование контроллеру.
   - Обновление состояния игры и отрисовка.
2. Реализовать главное меню согласно примеру из `rogue_sample`.
3. Обеспечить возможность паузы, сохранения, выхода.

## Этап 7: Дополнительные задания (Bonus)
1. **Задание 6**: Двери и ключи.
   - Генерация дверей между комнатами/коридорами.
   - Ключи разных цветов.
   - Проверка доступности ключей (алгоритм поиска).
2. **Задание 7**: Динамическая сложность.
   - Адаптация количества врагов/предметов на основе успехов игрока.
3. **Задание 8**: Мимник.
   - Новый тип врага, маскирующийся под предмет.
4. **Задание 9**: 3D рендеринг.
   - Режим первого лица с Ray Casting.
   - Текстуры стен, мини-карта в углу.
   - Управление WASD для движения и поворота.

## Этап 8: Тестирование и отладка
1. Написание unit-тестов для Domain Layer (покрытие >80%).
2. Интеграционные тесты для Presentation и Data слоёв.
3. Ручное тестирование игрового процесса, балансировка.
4. Исправление багов, оптимизация производительности.

## Этап 9: Документация и финализация
1. Написание README с инструкциями по сборке и запуску.
2. Комментарии в коде (godoc).
3. Создание демонстрационного видео/скриншотов.
4. Подготовка релиза (бинарные файлы для Linux/Mac/Windows).

## Ориентировочные сроки
- Этапы 1-3: 2 недели
- Этапы 4-6: 3 недели
- Этап 7 (bonus): 2 недели
- Этапы 8-9: 1 неделя

Итого: ~8 недель на полную реализацию.

## Риски и митигация
- Сложность реализации Ray Casting для тумана войны и 3D: использовать готовые алгоритмы из примеров.
- Балансировка игровых механик: итеративная настройка на основе тестов.
- Производительность при большом количестве сущностей: оптимизация структур данных.
